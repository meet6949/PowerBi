2. Schema Design
   ->Star Schema
     Sales_ Fact ko central fact table rakha gaya aur dimension tables uske around connect:
     Customer_ Dim → Sales_ Fact
     Product_ Dim → Sales_ Fact
     Region_ Dim → Sales_ Fact
     Date_ Dim → Sales_ Fact

   ->Cardinality Types
     1 : Many (One-to-Many)
     Ek table me unique value hoti hai aur dusri table me same value multiple records 
     Example: Customer_ Dim → Sales_ Fact

     Many : 1 (Many-to-One)
     Multiple records ek single record ko refer karte hain.
     Example: Sales_ Fact → Product_ Dim

     1 : 1 (One-to-One)
     Dono tables me key value sirf ek baar hoti hai.
     Example: Sales_ Fact → Returns_ Fact

3. Advanced Model Settings

   Relationship Settings
   Relationships ko One-to-Many rakha.
   Single cross-filter direction use kiya.
   Bidirectional Filters
   Default me disable rakhe.
   Sirf zarurat hone par enable kiye ja sakte hain.

   Inactive Relationship
   Date_ Dim aur Returns_ Fact ke beech inactive relationship 
   Calculation me USERELATIONSHIP() function 

   Filter Ambiguity
   Model check taaki multiple filter paths na ho.
   Single filtering se circular filtering avoid ki.
